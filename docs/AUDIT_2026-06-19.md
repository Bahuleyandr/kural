# Kural — Security & Code Audit (2026-06-19)

Merged findings from two independent read-only audits of the Kural monorepo
(backend FastAPI, Next.js frontend, Tauri desktop, CLI, MCP server) at
`main`-derived branch `fix/clone-runtime-torch`.

**Threat model:** privacy-first, **offline, single-tenant** desktop app, with an
explicitly-supported networked/LAN mode gated by `KURAL_API_KEY`, and an agentic
**MCP** surface. Single-tenant-by-design is intended, not a defect.

**Overall:** strong application-security fundamentals (no XSS sinks, path
traversal closed via UUID-normalization, `defusedxml`, all user regex
`re.escape`d, zero `shell=True`, constant-time auth, localhost-only sidecar with
a per-install key, digest-pinned containers). The residual risk has moved to the
**supply chain**, **model-pack trust**, the **agentic MCP surface**, and
**release assurance / LAN-mode hardening**.

Each item links the source location and tags its origin (audit A = broad
sweep / supply-chain + concurrency; audit B = release-assurance + ergonomics).

## Verified-clean (probed, no issue)

No XSS sinks anywhere in the frontend; path traversal closed by UUID +
`relative_to`; SSML via `defusedxml`; all user regex `re.escape`d; zero
`shell=True`; constant-time `hmac.compare_digest` auth shared HTTP+WS; Ollama
SSRF fenced to loopback; Tauri minimal capabilities + fail-closed updater pubkey
validation + real Authenticode verify; digest-pinned non-root containers; Kokoro
downloads SHA-256-pinned by default; no secrets committed.

## Remediation status

See branch `fix/audit-2026-06-19-remediation`. Items are fixed in tier order;
the per-stack local CI gate (`backend pytest`, `cli pytest`, `mcp pytest`,
`frontend vitest`, `cargo check --locked`) is the merge gate (GHA is
billing-blocked).
